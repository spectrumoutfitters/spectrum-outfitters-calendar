# HTTPS Setup for Camera Access

The application now uses HTTPS to enable camera access over the network. This is required because browsers only allow camera access in secure contexts (HTTPS or localhost).

## What Changed

- Vite dev server now runs with HTTPS enabled
- Access URLs changed from `http://` to `https://`
- Self-signed certificate is automatically generated by Vite

## Accessing the Application

### Network Access (from other devices)
```
https://192.168.1.100:5173
```
(Replace with your actual IP address)

### Local Access
```
https://localhost:5173
```

## Security Warning

When you first access the application via HTTPS, your browser will show a security warning because we're using a self-signed certificate. This is **normal and safe** for development.

### To Proceed:

**Chrome/Edge:**
1. Click "Advanced" or "Show Details"
2. Click "Proceed to [your-ip] (unsafe)" or "Continue to localhost"

**Firefox:**
1. Click "Advanced"
2. Click "Accept the Risk and Continue"

**Safari:**
1. Click "Show Details"
2. Click "visit this website"
3. Click "Visit Website" in the popup

## Why HTTPS is Required

Modern browsers require HTTPS (or localhost) for:
- Camera/microphone access
- Geolocation
- Service Workers
- Other secure features

Without HTTPS, you'll see the error: "Camera access is not available in this browser or context."

## Troubleshooting

### Certificate Issues
If you have certificate issues, you can:
1. Clear your browser cache
2. Try accessing from an incognito/private window
3. Restart the dev server

### Still Can't Access Camera?
1. Make sure you're using `https://` not `http://`
2. Accept the security warning
3. Grant camera permissions when prompted
4. Check that your device has a camera and it's not being used by another app

## Production

For production, you'll need a proper SSL certificate from a Certificate Authority (like Let's Encrypt). The self-signed certificate is only for development.
